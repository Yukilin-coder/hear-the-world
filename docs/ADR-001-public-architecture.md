# ADR-001：公开版采用 Supabase + 单体 Node Web 服务

## 状态

已采纳，2026-07-16。

## 背景

产品需要邮箱注册登录、个人生词本、个人故事、书签、句子收藏，以及按播客句段展开的公开评论。当前原型把数据保存在浏览器 `localStorage`，无法跨设备同步，也没有用户隔离。

## 决策

- 继续使用现有 Node 静态服务，避免过早拆分微服务。
- 使用 Supabase Auth 处理邮箱注册、登录和会话。
- 使用 Supabase Postgres 保存用户数据和公开评论。
- 使用 Row Level Security 确保用户只能读写自己的生词、故事、书签和收藏。
- 评论允许所有人读取，只有登录用户可以发布，且只能修改或删除自己的评论。
- 音频不跟随代码仓库部署。公开内容优先采用 YouTube 官方嵌入；自有或已授权音频可放在 Cloudflare R2。

## 取舍

优点：开发和运维简单、起步成本低、权限规则清晰，适合 MVP。  
缺点：依赖 Supabase；故事生成目前仍是规则模板，后续接入 AI 时需要增加服务端函数和用量控制。

## 失败与防护

- Supabase 不可用时，网页回退到本机游客数据。
- 未登录用户不能发表评论，避免匿名垃圾内容。
- 数据表全部启用 RLS，前端公开的 anon key 不具备绕过权限的能力。
- 评论限制为 1–1000 字；正式上线还应增加举报、限流和管理员审核。
